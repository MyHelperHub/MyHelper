桌面宠物Live2D渲染系统实现
基于Tauri + Vue3 + TypeScript项目，使用PIXI.js 8 + easy-live2d实现桌面宠物Live2D渲染，支持新窗口模式和小窗内嵌模式，为后续接入龙骨、Lottie等技术预留扩展接口。

文件结构
src-tauri/assets/models/
├── live2d/built-in/        # Live2D模型已放置
├── spine/built-in/         # 预留
└── lottie/built-in/        # 预留

src/desktop-pet/
├── types.ts               # 动画技术类型定义
├── detector.ts            # 模型类型检测
├── factory.ts             # 渲染器工厂
├── manager.ts             # 动画管理器
├── renderers/
│   ├── live2d.ts         # Live2D渲染器(使用easy-live2d)
│   ├── spine.ts          # 预留接口
│   └── lottie.ts         # 预留接口
├── components/
│   ├── PetWindow.vue     # 独立窗口宠物组件
│   └── PetInline.vue     # 内嵌小窗宠物组件
└── composables/
    └── usePet.ts         # 宠物状态管理
显示模式要求
1. 新窗口模式
创建独立的宠物窗口(透明、置顶、无边框)
窗口大小300x400，支持拖拽移动
使用现有WindowManager管理窗口
2. 小窗内嵌模式
替换现有小窗模式的logo显示
适配小窗尺寸，Live2D模型自适应缩放
保持小窗的其他交互功能不变
点击可切换到新窗口模式
核心要求
1. 技术类型检测
定义AnimationTechnology枚举(LIVE2D/SPINE/LOTTIE/UNKNOWN)
复用现有IPC通信机制，Rust端提供检测API
前端调用检测接口获取模型技术类型
2. 可扩展渲染架构
IAnimationRenderer统一接口
工厂模式根据技术类型创建渲染器
当前实现Live2DRenderer(使用easy-live2d库)，其他预留接口
支持不同尺寸的自适应渲染
3. 资源管理
复用现有GlobalData状态管理
通过asset://协议加载已放置的Live2D模型
使用现有错误处理和API响应机制
4. 状态管理
使用usePet composable管理宠物状态
集成现有的设置系统，支持宠物开关配置
记住用户选择的显示模式和当前模型
技术配置更新
依赖配置

Live2D渲染器实现要点
使用easy-live2d库替代pixi-live2d-display
适配PIXI.js 8版本的API变化
支持Live2D Cubism 4.0+模型格式
实现鼠标跟踪、动画播放、表情切换等功能
针对小窗和大窗不同尺寸优化渲染性能
开发规范要求
1. 高内聚低耦合
宠物功能独立封装在desktop-pet目录
渲染器之间相互独立，通过统一接口交互
显示模式组件职责单一，可独立使用
2. 复用现有工具
使用现有的IPC API封装(invokeApi、safeInvokeApi)
复用WindowManager创建和管理宠物窗口
使用现有的错误处理机制(ErrorHandler)
集成现有的权限系统和配置管理
3. 组件设计
PetInline组件适配小窗尺寸，自动缩放
使用usePet管理两种模式的状态同步
集成要求
在主窗口设置中添加宠物功能开关
小窗模式下替换logo为Live2D宠物
支持运行时切换显示模式
保持与现有功能的兼容性
扩展设计
新增动画技术只需：添加枚举值 → 实现检测规则 → 创建渲染器 → 工厂注册

特别注意
必须使用PIXI.js 8版本
使用easy-live2d库而非pixi-live2d-display
确保与PIXI.js 8的API兼容性
处理可能的版本差异和API变更
请基于现有项目架构和easy-live2d库实现完整功能，确保代码复用性和可维护性。
